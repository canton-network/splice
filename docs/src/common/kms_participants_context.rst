By default, Canton participants use cryptographic `keys <https://docs.canton.network/global-synchronizer/production-operations/kms-operations>`_ that are generated by the participant itself and stored in the regular database used by the participant.
To improve key security, participants can be configured to use an external Key Management System (KMS) to generate and store keys.
Please consult the official `Canton documentation on KMS support <https://docs.canton.network/global-synchronizer/production-operations/kms-operations>`_ for additional details and a list of supported KMS providers.
Participants deployed as part of a Splice deployment support the `External Key Storage <https://docs.canton.network/global-synchronizer/production-operations/kms-operations#enable-external-key-storage-with-a-kms>`_ mode of KMS usage.
